Every Developer output triggers Tester verification before DevOps proceeds.

## Non-negotiable principles ‚Äî apply to every role
1. **Own skeleton for every external provider**
   Project-owned interface in `Application/Abstractions/`. Concrete adapter in `Infrastructure/` only.
   Your interface defines the contract ‚Äî vendors adapt to you, never the reverse.
   Serilog uses `Log.Information`, NLog uses `logger.Info` ‚Äî your code calls `_logger.Info` on `IAppLogger<T>`.
   Changing providers = write new adapter + change one DI line. Zero Domain or Application changes.

2. **No comments** ‚Äî rename instead of explaining. No XML doc comments (`///`).

3. **No vendor types in Domain or Application** ‚Äî `ILogger<T>`, SDK clients, ORMs belong in Infrastructure only.

4. **Async throughout** ‚Äî `CancellationToken` on every async public method, never `.Result` or `.Wait()`.

5. **Dependency rule** ‚Äî dependencies point inward: Infrastructure ‚Üí Application ‚Üí Domain.

6. **No automatic git or GitHub actions** ‚Äî never commit, push, create branches, open PRs,
   post comments, or trigger pipelines without an explicit instruction. Produce the output
   and wait. All MCP write operations require a direct ask ‚Äî do not infer permission from context.

7. **Commit and comment style**
   - Commit messages: 2‚Äì5 words with conventional prefix ‚Äî `fix: null check`, `feat: add login`, `refactor: extract interface`
   - Group commits by small logical changes ‚Äî one concern per commit, never bundle unrelated changes
   - No Claude references anywhere ‚Äî no "Generated by Claude", no "AI:", no co-author tags, nothing identifying Claude as author
   - PR descriptions and review comments: same rules ‚Äî short, no Claude references

## GitHub integration (Engineering Manager)
GitHub issue creation is OFF by default.
To enable for a session: tell the Engineering Manager "create tickets on GitHub" or "enable GitHub flow".
To enable permanently: add "GitHub integration: ON" to this file.
To disable permanently: add "GitHub integration: DISABLED" to this file.

## Image dimension limit
Claude errors when images in the conversation context exceed 2000px on any dimension.
This most commonly happens when Developer or Tester uses Playwright MCP to debug
a running app and takes screenshots of it.

Rules that apply to every role:
- **Prefer `browser_snapshot`** (accessibility tree, no image) over `browser_screenshot` for inspecting structure, layout, or errors during debugging
- **Never use `fullPage: true`** on screenshots ‚Äî visible viewport only
- **Discard screenshots immediately after use** ‚Äî once a screenshot has been analysed, run `/compact` in Claude Code to remove it from context. Do not let images accumulate.
- **One at a time** ‚Äî analyse one screenshot, compact, then take another if still needed
- Playwright MCP viewport is locked at 1280√ó800 in `claude.json` ‚Äî this prevents most cases automatically
